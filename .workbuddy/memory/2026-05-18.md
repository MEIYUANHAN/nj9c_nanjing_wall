## 修复 gunicorn WORKER TIMEOUT（Railway 容器 sendfile 不兼容）

**问题现象**：
- 日志出现 `WORKER TIMEOUT (pid:2)` + `Worker was sent SIGKILL`
- 媒体文件请求（图片）出现 `Not Found: /media/static/cq1.jpeg`
- Traceback 指向 `gunicorn/http/wsgi.py sendfile → socket.sendfile → os_sendfile`

**根本原因**：
Railway 容器不支持 Linux `os.sendfile()` 系统调用。gunicorn 默认对文件响应启用 sendfile 优化，尝试调用时进程挂起，超过 timeout 后被 SIGKILL 杀死。worker 崩溃重启期间所有请求（含图片）都 404。

**修复**：
1. 新建 `gunicorn.conf.py`：`sendfile = False`（同时设 `workers=1`、`timeout=120`）
2. 修改 `Procfile`：添加 `--config gunicorn.conf.py`

**附注**：
- 数据库中图片路径是 `static/xxx`，文件在 `media/static/`，路径本身正确，404 全是 worker 崩溃引起的
